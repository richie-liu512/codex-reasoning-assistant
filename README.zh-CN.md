# Codex 智能推理助手

[English](README.md)

在每次任务开始时显示当前 Codex 模型和推理等级，并根据接下来的工作给出继续、升档、降档或更换模型的建议。

## 它能做什么

- 读取当前任务实际采用的模型和推理等级；
- 显示该等级在当前模型支持范围中的位置；
- 判断当前配置能否保证下一阶段的完成质量；
- 在长线任务的自然阶段边界重新给出建议；
- 无法确认实际等级时明确说明，不把配置默认值当成当前任务的实际值。

每次任务开始时，你会看到一条简短的检查结果，例如：

```text
推理检查：GPT-5.6 Sol · 超高（实际值，第 4/6 级）｜建议：Sol · 高｜继续执行
```

如果当前等级不足，助手会在进入高难度阶段前暂停，并说明已经完成的内容、剩余工作、建议配置和恢复位置。如果当前等级明显高于剩余工作的需要，助手会先完成真正需要较强推理的部分，再在合适的检查点建议降低。配置合适时，任务会直接继续。

## 它如何判断

助手先判断哪一种模型能够胜任任务，再选择满足质量要求的最低推理等级。判断主要考虑：

- 目标和证据是否存在歧义；
- 结论依赖的推理链是否较深或容易出错；
- 错误结果是否容易通过测试或复查发现；
- 失败的影响和可逆性；
- 工具、状态、并发和恢复过程是否复杂。

文件数量、任务长度和预计 Token 消耗不会被直接当成推理难度。不同模型上的同名推理等级也不代表相同能力。

模型定位和等级建议来自带复查日期的策略文件。当前依据和官方来源见 [推理策略](docs/reasoning-policy.md)。

## 安装

需要：

- 支持插件和生命周期 Hook 的 Codex；
- Python 3.10 或更高版本，无需安装第三方 Python 包。

添加插件市场：

```bash
codex plugin marketplace add richie-liu512/codex-reasoning-assistant --ref main
```

然后：

1. 打开 `/plugins`；
2. 选择 `codex-reasoning-assistant` 市场并安装 **Codex Reasoning Assistant（Codex 智能推理助手）**；
3. 打开 `/hooks`，检查并信任插件携带的 Hook；
4. 开启一个新任务。

如果你已经配置了用于检测推理等级的 `UserPromptSubmit` Hook，请先停用其中一个，避免重复显示检查结果。

## 使用

安装完成后，每次提交任务都会自动进行检查，不需要额外提示。

你也可以直接要求：

```text
检查当前模型和推理等级是否适合这个任务。
```

或者在长线任务的阶段边界询问：

```text
下一阶段建议使用什么模型和推理等级？
```

助手会给出建议；是否切换由你决定。只有在你明确要求时，它才会协助修改配置或启动新的任务。

## 当前值的可信度

- `actual（实际值）`：模型来自当前 Hook 事件，推理等级由当前任务状态确认；
- `default_only（仅默认值）`：只读取到 Codex 配置中的默认等级，尚未确认当前任务是否采用；
- `unavailable（无法确认）`：当前环境无法可靠读取实际等级。

模型目录中的默认等级会单独展示，不会被当成当前任务的实际等级。

## 数据与权限

检测过程只读取确认模型和推理等级所需的 Codex 元数据，不会输出、保存、记录或传输提示词正文，也不会写入 Codex 的任务状态。

详细说明见 [隐私说明](PRIVACY.md) 和 [安全说明](SECURITY.md)。

## 验证项目

克隆仓库后可运行：

```bash
python plugins/codex-reasoning-assistant/scripts/verify.py
```

验证内容包括插件结构、Hook 包装器、隐私扫描、Python 语法和单元测试。

## 兼容性

Codex 的插件格式和内部状态结构可能继续变化。读取失败时，任务仍会继续，助手会把当前等级标记为无法确认。
